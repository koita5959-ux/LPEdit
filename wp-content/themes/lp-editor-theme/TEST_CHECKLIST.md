# LP Editor 本番切替チェックリスト

## 使い方
- 1項目ずつ実施して `[ ]` を `[x]` に変更
- NG項目が1つでもあれば本番切替を止める
- 実施日・実施者を最後に記録
- 実施後は `TEST_REPORT_TEMPLATE.md` に結果を記録

## A. 基本導線
- [ ] `page-select` が表示される
- [ ] 「新しくLPを作成する」で `editor` に遷移する
- [ ] 「公開中のLPを編集する」からメール送信操作ができる

## B. 新規作成フロー
- [ ] `editor` で必須項目（電話・メール）未入力時に保存できない
- [ ] 保存後に `?edit=...&created=1` の編集URLへ遷移する
- [ ] 「公開ページを見る」で作成したLPが開く
- [ ] フロント表示に重大な崩れがない

## C. 編集フロー
- [ ] 発行された編集URLで編集画面を開ける
- [ ] 更新後に内容が公開ページへ反映される
- [ ] トークンなし更新APIは拒否される（403）

## D. 画像アップロード
- [ ] JPEG/PNG/GIF/WebP をアップロードできる
- [ ] 5MB超の画像が拒否される
- [ ] 連続アップロードでレート制限（429）が効く

## E. 編集URL発行API
- [ ] 同一メールで連続送信時にレート制限（429）が効く
- [ ] 429時に待機秒メッセージが表示される
- [ ] リロード後もクールダウン表示が維持される

## F. セキュリティ・安定性
- [ ] `preview/create/upload/request-edit-url` がNonceなしで拒否される
- [ ] `update/delete` が編集トークンなしで拒否される
- [ ] PHPエラーログに致命的エラーが出ていない

## G. メール
- [ ] 編集URLメールが送信される（本番）
- [ ] お問い合わせ送信で管理者通知メールが届く
- [ ] 自動返信メールが届く

## H. 最終確認
- [ ] 既存LPが閲覧・編集できる
- [ ] 管理画面で意図しない固定ページ/投稿更新が起きていない
- [ ] 主要画面の表示速度に大きな劣化がない
